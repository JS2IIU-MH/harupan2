# Harupan2

## 1. アプリ概要

某食品メーカー祭りのシールをカメラでリアルタイム認識し、シールの点数を自動計算して合計点を表示するモバイルアプリ。

- **アプリ表示名**：Harupan2
- **アプリアイコン**：/assets/icon/icon.png

## 2. 対象プラットフォーム

- iOS / Android（Flutter によるクロスプラットフォーム開発）

## 3. 技術スタック

| 区分 | 技術 |
|------|------|
| フレームワーク | Flutter |
| 物体検出モデル | YOLOv8（ONNX） |
| 推論エンジン | onnxruntime_flutter（オンデバイス推論） |
| 状態管理 | Riverpod |
| 多言語対応 | flutter_localizations（日本語、英語は後日追加） |

## 7. 画面構成

### 7.1 構成
- **1 スクリーン構成**（単一画面アプリ）
- アプリ起動時にカメラ ON → リアルタイム認識を即時開始

### 7.2 UI レイヤー（下から順）
1. **カメラプレビュー**（背面カメラ映像）
2. **バウンディングボックス**（検出されたシールごとにラベル・点数を表示）
3. **合計点表示**（画面中央に大きな文字で半透明表示、カメラ映像に重畳）
4. **操作 UI**（カメラ前面/背面切替ボタン、シェアボタン）

## 8. 主な機能

### 8.1 リアルタイムカメラ認識
- カメラ映像からシールをリアルタイムで検出・分類
- 検出されたシールにバウンディングボックスを描画
- 各シールの点数を即時表示
- 合計点数を自動計算して画面中央に半透明の大文字で表示
- **推論フレームレート目標**：約 10 fps
- **デフォルトカメラ**：背面カメラ
- **カメラ切替**：ユーザー操作で前面 / 背面カメラを切替可能

### 8.2 シェア機能（SNS 連携）
- スクリーンショット + 合計点 + ハッシュタグ `#harupan2` をシェア
- **OS 標準のシェアシート**を使用

## 12. 実機での不安定度定量評価（検証項目）

### 12.1 クラッシュ率

- **指標**
  - Crash-free sessions（%）
  - クラッシュ件数 / 1時間
  - ANR 率（Android）
- **条件**
  - 連続動作 30 分 / 60 分 / 120 分
  - 前面↔背面カメラ切替
  - バックグラウンド復帰
  - 権限拒否→許可
- **合格目安**
  - Crash-free sessions 99% 以上
  - ANR 0%

### 12.2 FPS（推論 + 表示）

- **指標**
  - 平均 FPS
  - P95 フレーム時間（ms）
  - 最低 FPS
- **条件**
  - 明所 / 暗所
  - 被写体あり / なし
  - 端末発熱後（開始 30 分後）
- **合格目安**
  - 平均 8〜10 fps 以上
  - P95 フレーム時間 200 ms 未満（目標 10 fps 運用）

### 12.3 温度

- **指標**
  - 端末表面温度
  - バッテリー温度
  - 温度上昇速度（℃ / 10 分）
- **条件**
  - 連続推論 60 分
  - 充電中 / 非充電
  - ケース有無
- **合格目安**
  - 危険温度帯に入らないこと
  - 端末警告・サーマルスロットリングの頻発なし

### 12.4 メモリ

- **指標**
  - RSS / PSS
  - Dart Heap
  - GC 頻度
  - メモリ増加傾向（MB / 10 分）
- **条件**
  - 連続 60 分
  - カメラ切替を定期実施
  - 共有機能を繰り返し実行
- **合格目安**
  - 時間経過で単調増加し続けない（リーク傾向なし）
  - OOM 0 件

### 12.5 テストマトリクス（最低限）

- 端末層: ロー / ミドル / ハイ 各 1 台以上（Android + iOS）
- OS: 現行 -1 を含む 2 世代以上
- 実行回数: 各シナリオ 3 回以上で平均・分散を記録

### 12.6 最終判定方法

- 上記 4 指標をスコア化（各 25 点、合計 100 点）
- 80 点未満: 不安定寄り
- 80〜90 点: 要改善
- 90 点以上: 実用安定
